Models

CGan with word embeddings


This is a conditional GAN that uses word embeddings as conditional input to the GAN. For the word embeddings we use 300 dimensional Glove word embeddings to encode captions that describe the emojis.

PCAGAN


For the PCA-GAN at first PCA is done on the whole training data, resulting in the corresponding eigenvectors. The Discriminator of the PCA-GAN is a usual discriminator just as in DCGAN. The generator on the other hand is a mapping network that maps the latent Z to n values, where n is the number of eigenvectors. The image is then generated by a linear combination of the eigenvectors using the values from the mapping network.

DCGAN


This approach uses a vanilla DCGAN as explained in the Pytorch Tutorial on DCGANs

Segment-GAN


For the Segment-GAN, the training data contains manually-annotated emoji parts as images. These parts fall into six classes: "ears", "eyebrows", "eyes", "hands", "mouth", and "tears". We build a Conditional GAN similar to CGAN. The difference is that we use one-hot encodings of the six classes as conditions instead of natural language text. The Conditional GAN is queried for each part specified by the user and then all the generated parts are pasted onto an image template which consists of a yellow circle on a black background.

Random Segments


There is no DGM behind that. We just pick a random image from our hand-annotated emoji parts for each user input. We then paste all the parts onto a background image, thus "generating" a new emoji.
